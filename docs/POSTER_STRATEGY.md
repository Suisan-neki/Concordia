# ポスター発表戦略

## 迷っている理由の分析

ポスター草案に15セクションあるが、ポスターは限られたスペースで**1つの核心**を伝えるべき。

### 現状の問題点

1. **何を主にしたいか不明確**
   - 思想か？技術か？実装か？
   - 医療特化か？汎用性か？

2. **「全部書かないと不安」という心理**
   - 15セクションは論文向け、ポスターには長すぎる
   - 聴衆は3分で読み切れない

3. **技術詳細と思想の混在**
   - セクション3で技術列挙、セクション2で脅威定義、セクション4でZPA定義
   - どれが主でどれが従か不明確

---

## ポスターの3つの選択肢

### 選択肢A: 「思想提案型」（哲学重視）

**主メッセージ**: 「合意は理解の軌跡である」

**構成**:
1. 問題提起: 同意書は圧力になる（1枚）
2. 視点転換: 理解の痕跡を記録する（1枚）
3. 実装例: Consent Lab（1枚）
4. 応用: 医療以外でも（1枚）

**見せ方**:
- 小さい台帳の画像
- タイムラインUIのスクリーンショット
- グループ写真 → 結婚 → 研究参加 の例

**技術は最小限**: Merkle chain の図だけ

**適する場面**: 概念発表、ワークショップ、セミナー

---

### 選択肢B: 「技術実装型」（セキュリティ重視）

**主メッセージ**: 「改ざん不能な合意トレース」

**構成**:
1. アーキテクチャ図: Capsule → Merkle → 検証（1枚）
2. 技術詳細: SHA-256, Ed25519, TSA（1枚）
3. 実装デモ: verify_chain.py の出力（1枚）
4. 応用: EHR vs Concordia（1枚）

**見せ方**:
- ハッシュチェーンの図
- 署名フローのシーケンス図
- 改ざん検出のデモ

**思想は最小限**: 「圧力ゼロ」の1行

**適する場面**: セキュリティ系カンファレンス、技術系イベント

---

### 選択肢C: 「問題解決型」（医療特化）

**主メッセージ**: 「医療の合意を圧力から解放する」

**構成**:
1. 医療現場の課題: 医師の恐れ + 患者の閉塞（1枚）
2. 解決策: 理解の痕跡記録（1枚）
3. 実装: Consent Lab の医療シナリオ（1枚）
4. 効果: トラブル予防、説明スキル向上（1枚）

**見せ方**:
- 診察室の写真（仮想）
- 医師/患者の声の引用
- セッション統計のグラフ

**技術は背景**: 「Merkle chain で改ざん検知」程度

**適する場面**: 医療IT系カンファレンス、患者安全イベント

---

## 推奨: 選択肢C（問題解決型）

**理由**:
1. **聴衆の共感が得やすい**: みんな医療を受ける
2. **問題が具体的**: 抽象的すぎない
3. **実装が主張を後押し**: Consent Lab が実際に動く
4. **広げやすい**: 「医療以外でも」で締めることで汎用性も示せる

**メッセージシート**:
```
タイトル: 医療における合意を「圧力」から解放する
副題: 理解の痕跡を改ざん不能に記録する仕組み

問題: 医師の恐れ、患者の閉塞
解決: 合意ではなく理解行動を記録
実装: Merkle chain + Ed25519 + Consent Lab
応用: 医療以外の合意形成でも
```

---

## ポスター構造（推奨）

### 左縦列（問題→解決）

**1. 医療現場の課題**
- 医師: 「疑われたらどうしよう」の恐れ
- 患者: 「断りづらい」閉塞感
- 図: 診察室での雰囲気（イラスト）

**2. 視点転換: 同意書から理解の痕跡へ**
- 従来: はい/いいえだけ
- Concordia: 説明→質問→再説明の往復を記録
- 図: UnderstandingEvent の時系列

### 右縦列（実装→応用）

**3. 技術実装**
- Merkle chain（改ざん検知）
- Ed25519 署名（不可否認性）
- Consent Lab（体験可能）
- 図: ハッシュチェーンの構造

**4. 医療以外の応用**
- グループ写真公開
- 不動産契約
- 結婚
- 図: 小→大のスケール

## 付録セクション（時間がある人向け）

**技詳細（小さめ）**:
- ZPA の定義（Voluntariness 等）
- 実装インバリアント（ハッシュ計算式）
- 段階導入ロードマップ

**体験方法（小さめ）**:
- QR コード → http://localhost:8000/lab
- 検証: `python scripts/verify_chain.py --session-id`
